# AWS Infrastructure Labs: Day 35 - Secure Private RDS & Web Server Integration

## 📋 Task Overview
The Nautilus DevOps team requires a secure, private relational database backend integrated with an existing front-end web application server. This hands-on lab demonstrates how to isolate a managed database instance inside a private ecosystem, configure secure cross-tier network firewall rules, establish passwordless administrative access via SSH, and configure a PHP runtime application connection string.

### Objectives
1. **Provision Private RDS Instance:** Launch a MySQL database named `devops-rds` using a Sandbox template on a `db.t3.micro` instance class with `gp2` storage type limited to `5GiB`.
2. **Initialize Schema:** Define the core initial database namespace as `devops_db` utilizing the administrative identity `devops_admin`.
3. **Firewall Rule Alignment (Security Groups):** * Expose standard HTTP outbound/inbound traffic on port `80` globally for the web server (`devops-ec2`).
   * Constrain ingress database access on port `3306` to listen strictly to traffic coming from the `devops-ec2` production node.
4. **Automated Key Exchange:** Create an RSA identification key matrix on the management platform (`aws-client`) and distribute the public key to the `devops-ec2` node for seamless passwordless terminal handshakes.
5. **Application Configuration:** Migrate a specialized `index.php` routing file into the Apache root layout and map the parameters to validate successful end-to-end cloud database communications.

## 🛠️ Step-by-Step Implementation Guide

### Step 1: Provision the Private RDS Database Engine
1. Navigate to the **Amazon RDS Dashboard** and select **Databases** -> **Create database**.
2. **Creation Method:** Choose **Standard create**.
3. **Engine Type:** Select **MySQL** with explicit version assignment **`8.4.5`**.
4. **Templates:** Select **Dev/Test (Sandbox)**.
5. **Settings:**
   * **DB instance identifier:** `devops-rds`
   * **Master username:** `devops_admin`
   * **Credentials management:** Choose *Self managed* and set a strong master password.
6. **Instance Configuration:** Choose **Burstable classes** and select **`db.t3.micro`**.
7. **Storage Allocation:**
   * **Storage type:** `General Purpose SSD (gp2)`
   * **Allocated storage:** `5` GiB
   * **Storage Autoscaling:** *Uncheck* "Enable storage autoscaling" to prevent size shifts.
8. **Connectivity Rules:**
   * **Public access:** Explicitly select **No** (maintains zero public exposure).
   * **VPC security group:** Choose *Create new* or link your dedicated group (e.g., `devops-rds-sg`).
9. **Additional Configuration:** Click to expand the panel and type **`devops_db`** inside the **Initial database name** box.
10. Click **Create database** and wait for the status to switch to a green **Available** state.

---

### Step 2: Configure Infrastructure Security Groups

#### Web Server Firewall Rules (`devops-ec2`)
1. Go to the **EC2 Console**, locate **`devops-ec2`**, and click its active Security Group link.
2. Edit the **Inbound Rules** to append:
   * **Type:** `HTTP` (Port 80) | **Source:** `Anywhere-IPv4` (`0.0.0.0/0`)
3. Save the configurations.

#### Database Backend Firewall Rules (`devops-rds-sg`)
1. Navigate to your RDS instance security profile configurations.
2. Edit the **Inbound Rules** to append:
   * **Type:** `MySQL/Aurora` (Port 3306) | **Source:** Choose **Custom** and input the explicit **Security Group ID** assigned to `devops-ec2`.
3. Save the modifications.

### Step 3: Configure Password-less Key-Based Authentication

Log into your **`aws-client`** administration terminal host to build and sync the secure key pair architecture.


# 1. Generate an identity key matrix if it does not exist on your host

```
if [ ! -f /root/.ssh/id_rsa ]; then
    ssh-keygen -t rsa -N "" -f /root/.ssh/id_rsa
fi
```

# 2. Extract and display the compiled public key string
```
cat /root/.ssh/id_rsa.pub
```
Copy the string output.
Pivot to the AWS Management Console and open a terminal interface onto your devops-ec2 instance using EC2 Instance Connect (acting as the root administrative operator).

Inject the authorization string into the target server profile:

```
Bash
mkdir -p /root/.ssh
chmod 700 /root/.ssh
echo "PASTE_THE_COPIED_PUBLIC_KEY_STRING_HERE" >> /root/.ssh/authorized_keys
chmod 600 /root/.ssh/authorized_keys
```

### Step 4: Deploy and Provision the Front-End Application Environment
Because the devops-ec2 web node runs on an Ubuntu operating platform distribution framework, initialize and verify the standard Apache web server and PHP packages using the Debian package management interface (apt).

Execute on the devops-ec2 node:

```
Bash
# Update the local indexes and install the Apache stack elements
sudo apt update -y
sudo apt install apache2 php php-mysql -y

# Fire up and register the daemon runtime layers
sudo systemctl start apache2
sudo systemctl enable apache2
```


Execute on the aws-client engine space:
With your passwordless configuration applied, securely migrate the source codebase file across endpoints. Replace <DEVOPS_EC2_PRIVATE_IP> with the internal IP address of your application instance.

```
Bash
scp /root/index.php root@<DEVOPS_EC2_PRIVATE_IP>:/var/www/html/index.php
```

Modify the Application String Config (On devops-ec2):

```
Bash
sudo vi /var/www/html/index.php
```

Locate the operational database engine connection criteria array blocks and swap in your unique platform specifications:

 - Host / Servername: Paste your RDS Endpoint string (e.g., devops-rds.xxxxxx.us-east-1.rds.amazonaws.com).
 - Username: devops_admin
 - Password: Use your custom DB password defined during step one.
 - Database / Dbname: devops_db

Save and write the file properties back into storage memory (:wq).

```
Bash
# Recycle the runtime process state engine to apply everything cleanly
sudo systemctl restart apache2
```
