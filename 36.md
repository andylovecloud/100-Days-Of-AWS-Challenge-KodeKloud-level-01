# AWS Infrastructure Labs: Day 36 - Application Load Balancer & Nginx Web Server

## 📋 Task Overview
The Nautilus Development Team needs to deploy a scalable, highly available web server infrastructure pattern. This hands-on lab demonstrates how to spin up a decoupled backend web application host, containerize a runtime script via EC2 user-data systems, and route structural client traffic utilizing an AWS Application Load Balancer (ALB) connected to a managed target group cluster.

### Objectives
1. **Firewall Isolation (Security Groups):**
   * Configure the infrastructure's **`default`** security group to open up inbound web traffic via port `80` to the public internet (`0.0.0.0/0`).
   * Provision a targeted **`datacenter-sg`** firewall boundary for the backend instance that handles traffic explicitly forwarded by the `default` security group layer.
2. **Launch EC2 Backend (`datacenter-ec2`):** Build an Ubuntu-based instance configured to provision an operational Nginx engine natively during initialization via boot strapping user-data mechanics.
3. **Establish Traffic Management Targets (`datacenter-tg`):** Provision a dedicated routing target group map to register the app server over port `80`.
4. **Deploy Application Load Balancer (`datacenter-alb`):** Orchestrate an internet-facing ALB tracking at least two Availability Zones to securely distribute external payload loops to the backend infrastructure.

---

## 🛠️ Step-by-Step Implementation Guide

### Step 1: Configure Network Security Groups

#### Part A: Update the Public ALB Firewall Rules
1. Navigate to the AWS Console -> **EC2 Dashboard** -> **Security Groups**.
2. Locate and check the box next to the **`default`** security group.
3. Click the **Inbound rules** tab -> select **Edit inbound rules**.
4. Append a rule mapping the following configuration values:
   * **Type:** `HTTP` (Port 80)
   * **Source:** `Anywhere-IPv4` (`0.0.0.0/0`)
5. Click **Save rules**.

#### Part B: Build the App Host Engine Firewall Rules
1. While still on the Security Groups interface, click **Create security group** (top-right).
2. Apply the structural configuration:
   * **Security group name:** `datacenter-sg`
   * **Description:** Security perimeter isolating backend ec2 nodes.
   * **VPC:** Choose your system's default Lab VPC.
3. Under the **Inbound rules** dashboard pane, click **Add rule**:
   * **Type:** `HTTP` (Port 80)
   * **Source:** Select **Custom** and search for/select the **`default` security group ID**.
4. Scroll to the bottom and select **Create security group**.

---

### Step 2: Establish the Load Balancer Target Group
1. On the left navigation column of the EC2 Dashboard, scroll to *Load Balancing* and select **Target Groups**.
2. Click **Create target group**.
3. **Choose a target type:** Select **Instances**.
4. Configure the parameters:
   * **Target group name:** `datacenter-tg`
   * **Protocol / Port:** `HTTP` / `80`
   * **VPC:** Ensure your system's default Lab VPC is selected.
5. Leave all additional advanced checks at factory settings. Click **Next**.
6. **Do not register targets manually yet.** Scroll past the interface lists and click **Create target group**.

---

### Step 3: Provision the Nginx Web Instance via User-Data Scripting
1. Navigate to the **EC2 Dashboard** -> **Instances** -> click **Launch instances**.
2. **Name:** `datacenter-ec2`
3. **Application and OS Images (AMI):** Choose **Ubuntu** (select any stable instance architecture version such as *Ubuntu 24.04 LTS* or *22.04 LTS*).
4. **Instance Type:** Select a baseline machine class (e.g., `t2.micro` or `t3.micro`).
5. **Network Settings:** Click **Edit** on the right side of the panel block.
   * **Firewall (security groups):** Select **Select existing security group**.
   * Check the tracking box for **`datacenter-sg`** and ensure the default group is left unselected.
6. **Advanced Details:** Expand this sub-menu pane and scroll to the bottom to locate the **User data** submission block. Copy and paste this structural configuration string cleanly:

```bash
#!/bin/bash
# Synchronize local advanced packaging engine metadata records
apt-get update -y

# Provision the Nginx web engine environment binaries natively
apt-get install nginx -y

# Boot up the processing engine and register it into startup runtime sequences
systemctl start nginx
systemctl enable nginx

```

Click Launch instance.

### Step 4: Deploy the Application Load Balancer

  1. On the left navigation column of the EC2 Dashboard, select Load Balancers.
  2. Click Create load balancer and choose Create under the Application Load Balancer engine template.
  3. Configure Basic Properties:
     - Load balancer name: datacenter-alb
     - Scheme: Internet-facing
     - IP address type: IPv4

  4. Network Mapping: Select your core Lab VPC and select at least two Availability Zones (along with an individual active public subnet mapping for each selected AZ area).
  5. Security Groups: Remove any automatic assignments and attach only the default security group.
  6. Listeners and Routing:
    - Protocol: HTTP | Port: 80
     Default action: Select Forward to -> and choose datacenter-tg.

  7. Click Create load balancer at the bottom of the page.

### Step 5: Register the Host Instance to the Target Infrastructure Pool

  1. Return to the Target Groups panel under the left tracking menu and select datacenter-tg.
  2. Navigate into the lower workspace tab layout and select Targets. Click Register targets.
  3. Locate the row tracking your active datacenter-ec2 node and select its checkbox.
  4. Ensure the routed routing port value parameter shows 80, then click Include as pending below.
  5. Finalize the pool attachment by selecting Register pending targets.

  2. Track the Health status indicator matrix. Wait roughly 2 minutes for the routing profile state checks to cycle from Initial over into a green Healthy confirmation check.
  3. Once the instance state updates successfully, pivot to the Load Balancers control grid and choose datacenter-alb.
  4. Locate the Details table panel block on the lower screen layout and copy the long DNS name tracking string completely.
