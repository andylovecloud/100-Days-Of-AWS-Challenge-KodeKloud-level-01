Day 33 : The Nautilus DevOps team is embracing serverless architecture by integrating AWS Lambda into their operational tasks. 
They have decided to deploy a simple Lambda function that will return a custom greeting to demonstrate serverless capabilities effectively. 
This function is crucial for showcasing rapid deployment and easy scalability features of AWS Lambda to the team.

- Create Lambda Function: Create a Lambda function named xfusion-lambda.
- Runtime: Use the Runtime Python.
- Deploy: The function should print the body Welcome to KKE AWS Labs!.
- Status Code: Ensure the status code is 200.
- IAM Role: Create and use the IAM role named lambda_execution_role.
- Use the AWS Console to complete this task. 


Steps:

Deploying a serverless AWS Lambda function on a custom IAM execution role is an efficient way to demonstrate rapid deployment pipelines and microservice architectures.

Here is the step-by-step implementation guide to create and configure your function via the AWS Management Console.

**Step 1: Create the IAM Execution Role**
AWS Lambda requires an execution role with permissions that allow it to write log statements to Amazon CloudWatch.

1. Open the AWS Management Console and navigate to the IAM Dashboard.
2. Click Roles in the left sidebar, then click Create role.
3. elect trusted entity: Choose AWS service, and from the service dropdown, select Lambda. Click Next.
4. Add permissions: Search for and select the checkbox next to AWSLambdaBasicExecutionRole. (This managed policy grants permissions to upload logs to Amazon CloudWatch). Click Next.
5. Role name: Type exactly lambda_execution_role.
6. Review the configuration and click Create role.


**Step 2: Create the Lambda Function (xfusion-lambda)**

1. Navigate to the AWS Lambda Dashboard and click Create function.
2. Choose Author from scratch.
3. Configure the Basic information pane:
  - Function name: xfusion-lambda
  - Runtime: Select the latest stable version of Python (e.g., Python 3.12 or Python 3.13).

4. Expand the Change default execution role section:
  - Select Use an existing role.
  - From the dropdown menu, select the lambda_execution_role you built in Step 1.

5. Click Create function at the bottom right.

**Step 3: Implement and Deploy the Code**
Once the Lambda console loads your function workspace, scroll down to the Code source editor tab.

1. Double-click lambda_function.py in the left-hand file explorer to load the editor window.
2. Replace the boilerplate code completely with the following compliant python script layout:

```
Python
import json

def lambda_handler(event, context):
    # Print the specific string to function logs
    print("Welcome to KKE AWS Labs!")
    
    # Return the structured payload matching the specifications
    return {
        'statusCode': 200,
        'body': json.dumps('Welcome to KKE AWS Labs!')
    }
```

3. Click the Deploy button located directly above the code editor window to save your functional changes into the active AWS runtime environment.

**Step 4: Validate the Function Setup**

To verify execution and assert the state requirements match:

1. Click the Test button right next to the Deploy button.
2. Create a new test event by entering an arbitrary Event name (e.g., testEvent), leaving the sample JSON payload as default, and clicking Save.
3. Click Test again to invoke the execution cycle.
4. Expand the Execution result banner. Ensure that:

  - Status: Displays Succeeded.
  - Response: Shows "statusCode": 200.
  - Function Logs: The output block includes the text string: Welcome to KKE AWS Labs!.
