# AWS Infrastructure Labs: Day 38 - End-to-End Containerization via ECR & ECS Fargate

## 📋 Task Overview
The Nautilus DevOps team is transitioning a local Python application into a fully managed container ecosystem. This hands-on lab demonstrates how to provision a private Amazon Elastic Container Registry (ECR), build and push an optimized Docker image from a local development host (`aws-client`), orchestrate a serverless Amazon Elastic Container Service (ECS) Fargate cluster, define runtime resource constraints using a Task Definition, and maintain application availability via an ECS Service.

### Objectives
1. **Provision Container Registry:** Create a private ECR repository named `datacenter-ecr` to securely house container images.
2. **Build and Publish Artifact:** Construct a Docker image from a designated workspace (`/root/pyapp`), bind the `latest` version tag, and push the artifact to the private ECR repository.
3. **Establish Compute Cluster:** Provision a serverless, high-availability ECS Cluster designated as `datacenter-cluster` built on AWS Fargate.
4. **Author Task Blueprint (`datacenter-taskdefinition`):** Map the published ECR Image URI into an isolated task profile restricting CPU, memory, and container port configurations.
5. **Orchestrate Cluster Service (`datacenter-service`):** Instantiate a persistent service layer with dedicated security group web access parameters to launch and maintain a minimum of one active application container task replica.

## 🛠️ Step-by-Step Implementation Guide

By following the [solution video](https://www.youtube.com/watch?v=7O90_guwipc)

### Step 1: Create the Private ECR Repository
1. Navigate to the AWS Console -> **Elastic Container Registry (ECR)** -> **Repositories** [00:01:49].
2. Click **Create repository** [00:02:01].
3. Configure Repository Settings:
   * **Visibility settings:** Select **Private**.
   * **Repository name:** Type exactly **`datacenter-ecr`**.
4. Leave all other parameters at their factory default configurations and click **Create repository** [00:02:10].
5. Select your newly created repository from the list, click **View push commands** [00:02:35], and copy the repository URI registry domain path.
> *Example Repository URI format:* `349855337166.dkr.ecr.us-east-1.amazonaws.com/datacenter-ecr`

---

### Step 2: Build, Tag, and Push the Docker Image

Log into your **`aws-client`** administration terminal host to authenticate your local container runtime engine with AWS and publish your application.

```bash
# 1. Navigate to the application workspace containing the target Dockerfile
cd /root/pyapp

# 2. Authenticate the local Docker client against your private ECR registry
aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 349855337166.dkr.ecr.us-east-1.amazonaws.com

# 3. Compile the container blueprint into a local image asset cleanly
docker build -t datacenter-ecr:latest .

# 4. Bind the formal ECR tracking tag descriptor mapping to the repository URI
docker tag datacenter-ecr:latest [349855337166.dkr.ecr.us-east-1.amazonaws.com/datacenter-ecr:latest](https://349855337166.dkr.ecr.us-east-1.amazonaws.com/datacenter-ecr:latest)

# 5. Ship the compiled artifact up to the cloud asset registry
docker push [349855337166.dkr.ecr.us-east-1.amazonaws.com/datacenter-ecr:latest](https://349855337166.dkr.ecr.us-east-1.amazonaws.com/datacenter-ecr:latest)
```


### Step 3: Provision the ECS Fargate Cluster

1. Navigate to the Elastic Container Service (ECS) Dashboard -> click Clusters in the left sidebar menu [00:04:32].
2. Click Create cluster [00:04:46].
3. Configure Cluster Properties:

 - Cluster name: Type exactly datacenter-cluster [00:04:51].
 - Infrastructure: Ensure AWS Fargate (serverless) is selected [00:04:57].

4. Click Create [00:05:07].

### Step 4: Author the ECS Task Definition

1. On the left navigation column of the ECS Dashboard, select Task definitions [00:05:18].
2. Click the Create new task definition dropdown and choose Create new task definition [00:05:18].
3. Configure the Task definition configuration properties:

 - Task definition family: Type exactly datacenter-taskdefinition [00:05:33].

4. Configure the Infrastructure requirements (Fargate launch specification):

 - Launch type: Select AWS Fargate [00:05:49].
 - Operating system/Architecture: Leave at default (Linux/X86_64) [00:05:55].
 - Task CPU: Select 0.5 vCPU [00:05:55].
 - Task memory: Select 1 GB [00:05:55].

5. Configure the Container - 1 configuration panel [00:06:07]:

 - Name: nginx-container [00:06:07]
 - Image URI: Click Browse ECR images, select datacenter-ecr, and pick the image tagged with latest [00:06:22]. (Alternatively paste: 349855337166.dkr.ecr.us-east-1.amazonaws.com/datacenter-ecr:latest)
 - Port mappings: Verify the port is configured explicitly to 80 with protocol TCP (matching the underlying exposed configuration) [00:06:33].
 - Essential container: Ensure this toggle is marked as Yes.

6. Scroll down to the bottom of the page interface and select Create [00:07:02].

### Step 5: Deploy the ECS Service

1. Navigate back to the ECS Clusters overview window pane and click into datacenter-cluster [00:07:37].
2. Under the Services sub-tab category workspace layout, select the Create button [00:07:37].
3. Configure the Environment settings:

 - Compute options: Choose Launch type [00:08:04].
 - Launch type: Select FARGATE [00:08:04].

4. Configure the Deployment configuration parameters:

 - Application type: Choose Service.
 - Family: Select datacenter-taskdefinition from the tracking dropdown list [00:07:49].
 - Revision: Choose the latest active value (e.g., 1 (latest)).
 - Service name: Type exactly datacenter-service [00:07:55].
 - Desired tasks: Type 1 [00:08:13].

5. Configure the Networking details [00:08:13]:

- VPC: Ensure your system default Lab VPC profile maps cleanly.
- Subnets: Select at least three active subnets (e.g., us-east-1a, us-east-1b, us-east-1c) [00:08:28].
- Security group: Select Create a new security group [00:08:37]:
  - Security group name: datacenter-service-sg
  - Inbound rules for security group: Click Add rule -> Type: HTTP | Port: 80 | Source: Anywhere-IPv4 (0.0.0.0/0) [00:08:51].

- Public IP: Ensure it is toggled to Turned on [00:08:59].

6. Scroll past Load Balancing matrices (leave at None) and click Create [00:08:59].
